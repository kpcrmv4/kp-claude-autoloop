How to use KP Claude Autoloop
What is this? Claude Code normally works only while you sit and chat with it. This program keeps it working for you: it wakes Claude up round after round until the job is finished — and when your Claude quota runs out, it simply waits and continues by itself when the quota comes back.
0 · First, have Claude write the work plan (once per project) This program doesn't think up the work — Claude needs a plan to follow. Open a Claude Code chat in your project, then copy the message below and paste it into that chat. Claude will create all the plan files for you: 
Create an autoloop workflow for this project (create the actual files):
1) docs/BUILD-PLAN.md — the full detailed plan, split into small work units, each with how to verify it
2) docs/BUILD-STATE.md — a "- [ ]" checklist matching the plan (the loop reads/ticks this file), and agree that when every item is done you append the line "AUTOLOOP: COMPLETE" at the end of the file
3) docs/round-prompt.txt — the per-round prompt: recover first (git status, finish any half-done work) → continue from BUILD-STATE.md, 3-5 work units per round → typecheck every unit, full test suite every ~3 units → commit locally, never push → update the checklist then end the turn immediately → when everything is done, append "AUTOLOOP: COMPLETE" to BUILD-STATE.md and reply with only AUTOLOOP: COMPLETE
4) docs/model-rules.json — {"default":{"model":"claude-sonnet-5","effort":"high"},"rules":[...]} with regexes matching the hard/risky plan items (e.g. money/tax/security) so they switch to claude-opus-4-8
When done, tell me this chat's session id.
1 · Come back here and pick that chat In the form below, open the Session dropdown and pick the chat you just used — the project folder fills in by itself. One rule: while the loop is running, don't open that chat and type into it yourself, or the two of you will step on each other's work.
2 · Press Start Once the project folder is set, the system looks for the plan files Claude created and fills in the rest of the form automatically. If it can't find them, you'll get a warning — that just means step 0 hasn't been done yet. Check the fields, press Start, and you can close this page — the work keeps going without it.
3 · Let it run — check in whenever you like Each job's card shows how much is done, what it's working on right now, and which AI model it's using. If your Claude quota runs out, you'll see a countdown to when it resumes — you don't need to do anything, it continues on its own. Want to stop? Press Stop — it finishes the current round cleanly first, so nothing breaks halfway.
When does it finish? When every item in the plan is done, Claude writes AUTOLOOP: COMPLETE at the end of the plan file and the loop stops by itself (with a Telegram message, if you've set that up). If it reaches the round limit (Max cycles) first, it pauses and waits for you to send it another batch.
Drive it from Telegram too Once the bot is set up (gear button above) and this dashboard is running, chat with your bot from your phone: /status shows a live card of every job with tap-buttons, /stop stops after the current job finishes, and /continue gives a job that hit its round cap a fresh budget to keep going.
โปรแกรมนี้คืออะไร? ปกติ Claude Code จะทำงานเฉพาะตอนเรานั่งคุยกับมัน โปรแกรมนี้ทำให้มันทำงานแทนเราต่อเนื่อง: คอยปลุก Claude ให้ลุกมาทำงานต่อทีละรอบจนงานเสร็จ และถ้าโควตา Claude หมดกลางทาง มันจะรอแล้วกลับมาทำต่อเองเมื่อโควตากลับมา ไม่ต้องเฝ้า
0 · ให้ Claude เขียนแผนงานก่อน (ทำครั้งเดียวต่อโปรเจกต์) โปรแกรมนี้ไม่ได้คิดงานเอง — Claude ต้องมีแผนให้เดินตาม วิธีทำ: เปิดแชท Claude Code ในโปรเจกต์ที่จะให้ทำ แล้วคัดลอกข้อความด้านล่างไปวางในแชทนั้น Claude จะสร้างไฟล์แผนงานให้ครบเอง: 
สร้าง workflow สำหรับรัน autoloop ในโปรเจกต์นี้ (สร้างไฟล์จริงให้เลย):
1) docs/BUILD-PLAN.md — แผนงานละเอียดทั้งหมด แตกเป็น work unit เล็ก ๆ พร้อมวิธี verify ของแต่ละข้อ
2) docs/BUILD-STATE.md — checklist "- [ ]" ตามแผน (ตัว loop จะอ่านต่อ/ติ๊กจากไฟล์นี้) และตกลงกันว่า งานครบทุกข้อ = เติมบรรทัด "AUTOLOOP: COMPLETE" ท้ายไฟล์
3) docs/round-prompt.txt — prompt ประจำรอบ: กู้ซากก่อน (git status เก็บงานค้างให้จบ) → ทำต่อจาก BUILD-STATE.md รอบละ 3-5 work unit → typecheck ทุก unit, test เต็มชุดทุก ~3 unit → commit local ห้าม push → อัปเดต checklist แล้วจบเทิร์นทันที → ถ้าครบทุกข้อ เติม "AUTOLOOP: COMPLETE" ท้าย BUILD-STATE.md และตอบแค่ AUTOLOOP: COMPLETE
4) docs/model-rules.json — {"default":{"model":"claude-sonnet-5","effort":"high"},"rules":[...]} โดยเขียน regex จับข้อที่ยาก/เสี่ยงในแผน (เช่น เงิน/ภาษี/security) ให้สลับเป็น claude-opus-4-8
เสร็จแล้วบอก session id ของแชทนี้มาด้วย
1 · กลับมาหน้านี้ แล้วเลือกแชทที่เพิ่งคุย ในฟอร์มด้านล่าง เปิดช่อง Session แล้วเลือกแชทตัวที่เพิ่งสั่งไปเมื่อกี้ — ช่องโฟลเดอร์โปรเจกต์จะถูกเติมให้เอง ข้อห้ามข้อเดียว: ระหว่างที่มันกำลังทำงาน อย่าเปิดแชทตัวนั้นเข้าไปพิมพ์คุยเอง เดี๋ยวงานตีกัน
2 · กดปุ่มเริ่มทำงาน พอใส่โฟลเดอร์โปรเจกต์แล้ว ระบบจะหาไฟล์แผนงานที่ Claude สร้างไว้ แล้วเติมช่องที่เหลือให้เองอัตโนมัติ ถ้าหาไม่เจอจะมีข้อความเตือน — แปลว่ายังไม่ได้ทำขั้น 0 ตรวจช่องให้ครบ กด เริ่มทำงาน เสร็จแล้วปิดหน้านี้ทิ้งได้เลย งานไม่หยุดตาม
3 · ปล่อยมันทำ แล้วแวะมาดูเมื่อไหร่ก็ได้ การ์ดของแต่ละงานจะบอกว่าเสร็จไปกี่ข้อจากทั้งหมด ตอนนี้กำลังทำข้อไหน และใช้ AI ตัวไหนอยู่ ถ้าโควตา Claude หมด จะเห็นตัวนับถอยหลังว่าจะกลับมาทำต่อกี่โมง — ไม่ต้องทำอะไรทั้งนั้น มันไปต่อเอง อยากหยุดก็กด หยุด — มันจะเก็บงานรอบที่ค้างให้จบเรียบร้อยก่อนค่อยหยุด งานไม่พังกลางทาง
แล้วมันจบเมื่อไหร่? เมื่องานในแผนเสร็จครบทุกข้อ Claude จะเขียนคำว่า AUTOLOOP: COMPLETE ปิดท้ายไฟล์แผน แล้วโปรแกรมก็หยุดเอง (พร้อมแจ้งเข้า Telegram ถ้าตั้งไว้) หรือถ้าทำครบจำนวนรอบที่ตั้งไว้ (Max cycles) ก่อนงานเสร็จ มันจะพักรอให้เรามาสั่งรอบเพิ่ม
สั่งงานจากมือถือผ่าน Telegram ได้ด้วย ตั้งค่าบอทแล้ว (ปุ่มเฟืองด้านบน) และหน้า dashboard นี้เปิดอยู่ ทักบอทจากมือถือได้เลย: /status ดูการ์ดสถานะสดของทุกงานพร้อมปุ่มกดสั่ง · /stop สั่งหยุดแบบรอจบงานปัจจุบันก่อน · /continue งานที่ครบรอบแล้วให้ทำต่อด้วยงบรอบชุดใหม่
